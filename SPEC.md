# NoteBase - 智能文章采集与清洗工具

## 1. 项目概述

- **项目名称**: NoteBase
- **项目类型**: Web 应用 (Node.js + 前端)
- **核心功能**: 输入网址自动采集网页内容，通过可视化操作删除不需要的部分，输出干净的学习文档
- **目标用户**: 需要从网页提取干净内容进行学习、笔记的用户

## 3. 功能与交互

### 3.1 核心功能

**URL 抓取**:
1. 用户在输入框输入 URL
2. 点击"抓取"按钮或按 Enter
3. 显示 loading 状态 (按钮显示加载动画)
4. 成功后: 左侧面板显示原始 HTML 内容
5. 失败时: 显示错误提示 toast

**可视化内容选择与删除**:
1. 鼠标悬停在左侧面板的可删除元素上时:
   - 元素显示红色虚线边框
   - 右上角出现删除图标
2. 点击元素:
   - 元素标记为选中状态 (红色实线边框 + 浅红背景)
   - 右侧面板实时更新，删除的内容同步消失
3. 再次点击已选中元素: 取消选中
4. 点击"删除选中"按钮: 确认删除所有选中元素

**撤销功能**:
- 记录操作历史
- 支持连续撤销 (Ctrl+Z / Cmd+Z)
- 最多保存 50 步历史

**导出功能**:
- 导出为 Markdown: 转换为标准 Markdown 格式
- 导出为 HTML: 干净的 HTML 片段
- 复制: 复制内容到剪贴板
- 下载: 下载为 .md 或 .html 文件
- HTML 导出可选「保留样式」，下载时包含基础 CSS 样式表

### 3.2 交互细节

**空状态**:
- 左侧面板: 显示引导文字 "输入网址开始抓取内容"
- 右侧面板: 显示引导文字 "在左侧选择要删除的内容"

**加载状态**:
- 按钮显示旋转加载图标
- 面板显示骨架屏或加载动画

**错误处理**:
- URL 格式错误: 输入框下方红色提示
- 网络错误: Toast 提示 "无法访问该网址，请检查 URL 或网络连接"
- CORS 错误: 后端自动处理，显示友好错误

**键盘快捷键**:
- Enter: 触发抓取 (焦点在输入框时)
- Ctrl+Z / Cmd+Z: 撤销
- Delete / Backspace: 删除选中元素 (有选中时)

## 4. 技术方案

### 4.1 技术栈

- **后端**: Node.js + Express
- **前端**: 原生 HTML/CSS/JavaScript (无框架依赖)
- **HTTP 客户端**: axios (后端 fetch)
- **HTML 解析**: cheerio
- **Markdown 转换**: turndown

### 4.2 项目结构

```
NoteBase/
├── server.js           # Express 服务器
├── package.json
├── public/
│   ├── index.html       # 主页面
│   ├── styles.css       # 样式文件
│   └── app.js           # 前端逻辑
└── SPEC.md
```

### 4.3 API 设计

**POST /api/fetch**
- 请求体: `{ url: string, keepImages?: boolean }` (keepImages 默认为 true)
- 成功响应: `{ success: true, title: string, content: string, originalHtml: string }`
- 失败响应: `{ success: false, error: string }`

### 4.4 数据流

1. 用户输入 URL → POST /api/fetch
2. 服务器使用 axios 获取页面 → cheerio 解析 → 返回结构化 HTML
3. 前端渲染原始内容到左侧面板
4. 用户点击删除 → 更新状态 → 右侧面板实时预览
5. 导出时调用 turndown 转换 → 返回 Markdown/HTML
